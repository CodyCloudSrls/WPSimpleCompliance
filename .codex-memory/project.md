# Project Memory

## Overview
WPSimpleCompliance is a WordPress plugin for EU-oriented cookie consent, privacy/cookie policy generation, cookie scanning, Google Consent Mode v2 defaults, and an accessibility statement helper.

The public GitHub repository is `CodyCloudSrls/WPSimpleCompliance`.

The public product name is `WPSimpleCompliance`. The historical WordPress plugin folder/file slug remains `simple-privacy-cookie-policy` to preserve update compatibility with existing installs.

## Architecture
- Main plugin entrypoint: `simple-privacy-cookie-policy.php`.
- Legal document generator: `includes/legal-generator.php`.
- Frontend consent behavior: `assets/simple-privacy-cookie-policy.js`.
- Frontend/admin styles: `assets/simple-privacy-cookie-policy.css` and `assets/simple-privacy-cookie-policy-admin.css`.
- Managed policy page template: `templates/policy-document.php`.
- GitHub Releases updater is implemented directly in the plugin, without an external update-checker dependency.
- The updater reads the latest public release from `CodyCloudSrls/WPSimpleCompliance` and expects a release asset named `simple-privacy-cookie-policy.zip`.

## Commands
- Build local package for inspection only:
  `rm -rf build dist && mkdir -p build/simple-privacy-cookie-policy dist && rsync -a ./ build/simple-privacy-cookie-policy/ --exclude '.git' --exclude '.github' --exclude '.codex-memory' --exclude '.gitattributes' --exclude '.gitignore' --exclude 'build' --exclude 'dist' --exclude '.DS_Store' --exclude '*.zip' && cd build && zip -qr ../dist/simple-privacy-cookie-policy.zip simple-privacy-cookie-policy`
- Release flow after GitHub auth is configured:
  `git push origin main && git push origin v1.2.0`
- The GitHub Actions release workflow regenerates the ZIP on tag pushes.

## Conventions
- Do not commit generated ZIPs or `dist/`; release ZIPs must be generated by GitHub Actions.
- Keep the plugin root folder in release archives as `simple-privacy-cookie-policy`.
- Keep user-provided repository license as AGPL-3.0.
- Avoid storing secrets, deploy keys, tokens, customer data, or generated release artifacts in memory.

## Deployment
- Repository remote: `git@github.com:CodyCloudSrls/WPSimpleCompliance.git`.
- A dedicated SSH key was generated locally at `~/.ssh/wpsimplecompliance_ed25519`; only the public key should be added to GitHub as a deploy key with write access.
- Local commit prepared for first plugin code push: `fc5f91c Prepare WPSimpleCompliance release workflow`.
- Local release tag prepared: `v1.2.0`.

## Known Risks
- The plugin cannot prove legal compliance by itself; final compliance depends on real site configuration, legal review, and blocking non-essential tags before consent.
- Local environment did not have `php` or `node`, so PHP/JS syntax checks were not run locally in the initial preparation session.
- GitHub push was blocked until SSH deploy key/write authentication is configured.
